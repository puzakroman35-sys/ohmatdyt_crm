# BE-203: Перепризначення кейсу (ADMIN)

Фаза: 2
Тип: Backend

## Мета
Надати дію для зміни відповідального виконавця з валідатором доступу до категорії та нотифікаціями.

## Скоуп
- POST /cases/{public_id}/reassign { responsible_id }
- Перевірити, що новий виконавець має доступ до категорії
- Розіслати листи старому/новому виконавцю

## Визначення готовності (DoD)
- Історія фіксує зміну відповідального

## Тести
- Валідні/невалідні перепризначення

## Залежності
- BE-006, BE-013, BE-014

## Позначки
- Reassign, Notifications
