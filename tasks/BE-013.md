# BE-013: Інтеграція Celery/Redis: воркер, розсилка листів, ретраї, лог нотифікацій

Фаза: 1 (MVP)
Тип: Backend

## Мета
Налаштувати Celery з Redis для асинхронної обробки нотифікацій, підтримати ретраї та логування відправок.

## Скоуп
- Celery worker + конфігурація брокера (Redis)
- Базові таски: send_email(payload)
- Ретраї з експоненційною затримкою (max_retries=5)
- Модель/таблиця NotificationLog (за потреби) для трекінгу

## Визначення готовності (DoD)
- Воркер стартує та приймає таски
- Листи відправляються через SMTP (BE-014)
- Помилки відправки ретраяться, статус фіксується

## Тести
- Юніт тести тасок; інтеграційний тест через Redis з моканим SMTP

## Залежності
- BE-001, BE-014

## Позначки
- Celery, Redis, Notifications
