# BE-006: Створення звернення (multipart) + email-тригер

Фаза: 1 (MVP)
Тип: Backend

## Мета
Реалізувати ендпоінт створення звернення оператором з можливістю завантаження файлів та тригером нотифікації для виконавців категорії.

## Скоуп
- POST /cases (multipart): { category_id, channel_id, applicant_name, applicant_phone?, applicant_email?, subcategory?, summary, files[] }
- Валідації обов’язкових полів, дозволених типів файлів
- Статус за замовчуванням: NEW
- Celery-тригер на відправку email виконавцям категорії

## Визначення готовності (DoD)
- Успішне створення повертає { public_id, status=NEW, ... }
- Файли прикріплюються і зберігаються з валідацією
- Нотифікація ставиться у чергу ≤ 1 хв після створення

## Тести
- Happy-path створення з 1–2 файлами
- Валідаційні помилки: відсутні обов’язкові поля, недопустимий файл

## Залежності

## Позначки
- Cases, Upload, Notifications, Celery
