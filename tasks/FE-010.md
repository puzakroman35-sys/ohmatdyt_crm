# FE-010: Додавання коментарів до звернення

Фаза: 1 (MVP)
Тип: Frontend

## Мета
Надати інтерфейс для додавання публічних та внутрішніх коментарів до звернення з урахуванням RBAC правил.

## Скоуп
- Компонент форми додавання коментаря на сторінці `/cases/[public_id]`
- Текстове поле (textarea) для введення коментаря
- Перемикач типу коментаря (публічний/внутрішній) - доступний лише для EXECUTOR та ADMIN
- Валідація:
  - Мінімум 5 символів
  - Максимум 5000 символів
- POST до `/cases/{public_id}/comments` з параметрами { text, is_internal }
- Оновлення списку коментарів після успішного додавання
- Блокування внутрішніх коментарів для OPERATOR ролі
- Повідомлення про успіх/помилку

## Визначення готовності (DoD)
- Форма працює згідно RBAC правил (OPERATOR не може створювати внутрішні коментарі)
- Після додавання коментар з'являється в списку без перезавантаження сторінки
- Валідації працюють коректно
- Помилки API відображаються користувачу

## Тести
- Додавання публічного коментаря (всі ролі)
- Додавання внутрішнього коментаря (EXECUTOR/ADMIN)
- Спроба додавання внутрішнього коментаря оператором (має бути заборонено)
- Валідація довжини тексту
- Відображення помилок
