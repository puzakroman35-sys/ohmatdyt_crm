# FE-011: Розширені права адміністратора для керування зверненнями

Фаза: 1 (MVP)
Тип: Frontend

## Мета
Реалізувати повний інтерфейс для адміністратора з можливістю зміни статусів будь-яких звернень, редагування полів звернення та призначення відповідальних.

## Скоуп
- Повний доступ до зміни статусів:
  - ADMIN може змінювати статус будь-якого звернення незалежно від відповідального
  - Доступні всі переходи статусів (Нове → В роботі → Очікує відповіді/Вирішено/Закрито)
  - Можливість повернення звернення в статус "Нове" з обнуленням відповідального
  - Обов'язковий коментар при зміні статусу

- Редагування полів звернення:
  - Редагування категорії та підкатегорії
  - Редагування каналу звернення
  - Редагування контактних даних (ім'я, телефон, email)
  - Редагування суті звернення
  - Модальне вікно або inline редагування з валідацією

- Управління відповідальним:
  - Можливість призначити/змінити відповідального виконавця
  - Відображення списку доступних виконавців
  - Можливість зняти відповідального (повернути звернення в пул)
  - Автоматичне повідомлення виконавцю про призначення

- Візуальна індикація прав:
  - Відображення всіх кнопок дій для адміністратора
  - Іконка "Редагувати" біля полів, які можна змінити
  - Відсутність обмежень на перегляд та зміну звернень

- Валідації та повідомлення:
  - Валідація всіх полів при редагуванні (як при створенні)
  - Підтвердження при критичних змінах (зміна категорії, зняття відповідального)
  - Повідомлення про успішне збереження змін
  - Відображення історії всіх змін

## Визначення готовності (DoD)
- ADMIN бачить всі звернення в системі без обмежень
- ADMIN може змінювати статус будь-якого звернення
- ADMIN може редагувати всі поля звернення через зручний UI
- ADMIN може призначати/знімати відповідальних виконавців
- Всі зміни валідуються та логуються
- UI чітко показує доступні дії адміністратора
- Відображаються success/error повідомлення для всіх операцій

## Тести
- ADMIN бачить всі звернення в системі
- ADMIN може змінити статус нового звернення на будь-який
- ADMIN може змінити статус звернення іншого виконавця
- ADMIN може редагувати поля звернення (категорія, канал, контакти, суть)
- ADMIN може призначити виконавця на нове звернення
- ADMIN може змінити відповідального виконавця
- ADMIN може зняти відповідального (повернути в статус "Нове")
- Валідації працюють при редагуванні полів
- Відображення помилок при невалідних даних
- Історія змін зберігається та відображається
