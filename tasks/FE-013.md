# FE-013: Фільтрація звернень для виконавців по категоріях

Фаза: 1 (MVP - розширення)
Тип: Frontend

## Мета
Адаптувати інтерфейс списку та деталей звернень для відображення тільки звернень з дозволених категорій для виконавців.

## Скоуп
- Модифікація сторінки списку звернень (`/cases`):
  - Автоматична фільтрація звернень для EXECUTOR на основі доступів до категорій
  - Відображення повідомлення, якщо у виконавця немає доступів до жодної категорії
  - Повідомлення: "У вас немає доступу до категорій. Зверніться до адміністратора."
  
- Модифікація фільтрів:
  - Для EXECUTOR у фільтрі категорій показувати тільки доступні категорії
  - Для ADMIN та OPERATOR - всі категорії
  - API запит для отримання доступних категорій виконавця
  
- Обробка помилок доступу:
  - При спробі відкрити звернення з недоступної категорії (403):
    - Відображення повідомлення: "У вас немає доступу до цього звернення"
    - Редирект на сторінку списку звернень
  - При спробі змінити статус звернення з недоступної категорії (403):
    - Відображення помилки: "У вас немає доступу до категорії цього звернення"
  
- Індикатор доступів для виконавця:
  - Відображення списку доступних категорій у профілі або сайдбарі
  - Бейдж з кількістю доступних категорій
  
- API інтеграція:
  - GET /users/me/category-access - отримати доступні категорії поточного користувача (якщо EXECUTOR)
  - Використання існуючих ендпоінтів GET /cases та GET /cases/{public_id}

## Визначення готовності (DoD)
- EXECUTOR бачить тільки звернення з доступних категорій
- У фільтрі категорій для EXECUTOR відображаються тільки доступні категорії
- При відсутності доступів відображається відповідне повідомлення
- При спробі доступу до недоступного звернення - коректна обробка 403
- ADMIN та OPERATOR не мають обмежень (бачать все)
- Помилки доступу відображаються зрозуміло для користувача
- Індикатор доступних категорій працює

## Тести
- EXECUTOR з доступом до категорії бачить звернення цієї категорії
- EXECUTOR не бачить звернення з недоступних категорій
- EXECUTOR без доступів бачить повідомлення про відсутність доступів
- Фільтр категорій для EXECUTOR показує тільки доступні категорії
- EXECUTOR намагається відкрити звернення з недоступної категорії (редирект + повідомлення)
- EXECUTOR намагається змінити статус недоступного звернення (помилка)
- ADMIN бачить всі звернення та всі категорії
- OPERATOR бачить всі звернення та всі категорії
- Індикатор доступних категорій відображається коректно
- Повідомлення про помилки доступу зрозумілі
