# BE-010: Зміна статусу (IN_PROGRESS -> NEEDS_INFO|REJECTED|DONE)

Фаза: 1 (MVP)
Тип: Backend

## Мета
Реалізувати зміну статусів для відповідального виконавця з обов’язковим коментарем і нотифікаціями.

## Скоуп
- POST /cases/{public_id}/status
  - Body: { to_status: IN_PROGRESS|NEEDS_INFO|REJECTED|DONE, comment }
  - Доступ: лише відповідальний EXECUTOR
- Історія в CaseStatusHistory (from->to, comment, author)
- Email оператору-автору
- Блокування редагувань після DONE/REJECTED (крім коментарів)

## Визначення готовності (DoD)
- Перевірка прав і валідності переходів
- Нотифікація ставиться у чергу

## Тести
- Валідні/невалідні переходи; обов’язковий коментар; RBAC

## Залежності
- BE-006, BE-013, BE-014

## Позначки
- Status Transitions, History, Notifications
