# BE-001: Ініціалізація бекенду та кастомний User

Фаза: 1 (MVP)
Тип: Backend

## Мета
Ініціалізувати Django-проєкт (ASGI), підключити PostgreSQL, створити кастомну модель користувача на базі AbstractUser з ролями та базовими полями.

## Скоуп
- Django 5+ проєкт (`api`)
- ASGI (Uvicorn/uvicorn[standard])
- Кастомний `User` з полями: username (унік.), email (унік.), full_name, role (OPERATOR|EXECUTOR|ADMIN), is_active
- Міграції БД, підключення PostgreSQL
- Адмін-панель: перегляд/редагування користувачів
- Фікстура суперкористувача (dev)

## Необхідні зміни БД
- Створення таблиці користувачів (кастомний User)
- Індекси на username, email, role

## Вимоги/обмеження
- Унікальність username та email
- Пароль-полісі: мін. 8 символів (літери+цифри)

## Визначення готовності (DoD)
- Проєкт стартує локально з PostgreSQL
- Створено суперюзера, логін в адмін-панель працює
- Модель User з потрібними полями й ролями, міграції застосовуються

## Тести
- Створення користувача, валідація унікальності email/username
- Перевірка ролей (enum)

## Залежності
- Немає

## Позначки
- DB, User, Roles, Django
