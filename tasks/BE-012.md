# BE-012: Управління користувачами (ADMIN): create/update, reset-password, deactivate

Фаза: 1 (MVP)
Тип: Backend

## Мета
Надати адміністратору можливість створення/редагування користувачів, скидання пароля та деактивації з перевіркою активних кейсів.

## Скоуп
- Ендпоінти (ADMIN):
  - GET /users (фільтри: role, is_active; пагінація/сортування)
  - POST /users { username, full_name, email, password, role, executor_category_ids?[] }
  - GET /users/{id}
  - PUT/PATCH /users/{id} { full_name, email, role, executor_category_ids }
  - POST /users/{id}/reset-password → { temp_password }
  - POST /users/{id}/deactivate (з перевіркою активних кейсів)
- Перевірка: деактивація EXECUTOR заборонена, якщо є кейси IN_PROGRESS|NEEDS_INFO; повернути 409 зі списком

## Визначення готовності (DoD)
- Операції працюють згідно RBAC
- Валідації ролі та категорій виконавця
- Тимчасовий пароль генерується і відправляється через Celery

## Тести
- Створення/редагування; reset-password; deactivate з/без активних кейсів
