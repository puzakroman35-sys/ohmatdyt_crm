# FE-002: Аутентифікація: логін, токени, guard’и

Фаза: 1 (MVP)
Тип: Frontend

## Мета
Реалізувати форму логіну та збереження/оновлення JWT, захист маршрутів за ролями (guard/ACL).

## Скоуп
- Сторінка /login з формою (username, password), валідація обов’язкових полів
- Виклик API: POST /auth/login; збереження access/refresh (httpOnly-cookie або пам’ять + refresh-флоу)
- Перехоплювачі запитів (refresh токена при 401, якщо потрібно)
- Guard компонент/ХОС: редіректи на /login для неавторизованих; перевірка ролей

## Визначення готовності (DoD)
- Успішний логін редіректить на домашню сторінку відповідно до ролі
- Закриті маршрути недоступні без авторизації

## Тести
- Успішний/неуспішний логін, редіректи, захист маршрутів
