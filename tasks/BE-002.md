# BE-002: Аутентифікація JWT (login/refresh/logout), CORS/CSRF

Фаза: 1 (MVP)
Тип: Backend

## Мета
Налаштувати JWT-аутентифікацію (access/refresh), ендпоінти входу/оновлення/виходу, базовий CORS та CSRF підхід для httpOnly-cookie (опц.).

## Скоуп
- Пакет для JWT (наприклад, `django-ninja-jwt` або власна імплементація)
- Ендпоінти: POST /auth/login, POST /auth/refresh, POST /auth/logout (опц.)
- Конфігурація тривалості access (15–30 хв) і refresh (7–30 днів)
- Підтримка Bearer-токена та/або httpOnly-cookie
- CORS: дозволені origins з env

## Контракти API
- POST /auth/login { username, password } -> { access, refresh, user }
- POST /auth/refresh { refresh } -> { access }

## Визначення готовності (DoD)
- Успішний логін повертає access/refresh та дані користувача
- Захист приватних ресурсів працює (401/403)
- CORS-конфіг керується змінними середовища

## Тести
- Успішний і неуспішний логін
- Оновлення токена
- Доступ до приватних ендпоінтів з/без токена
