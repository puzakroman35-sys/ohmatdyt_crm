# FE-012: UI управління доступами виконавців до категорій

Фаза: 1 (MVP - розширення)
Тип: Frontend

## Мета
Розширити розділ управління користувачами для адміністратора з можливістю управління доступами виконавців до категорій.

## Скоуп
- Розширення сторінки редагування користувача (`/admin/users/[id]/edit`):
  - Секція "Доступ до категорій" (тільки для виконавців)
  - Компонент вибору категорій (Transfer або Checkbox Group з AntD)
  - Ліва колонка: доступні категорії
  - Права колонка: обрані категорії
  - Можливість пошуку категорій
  - Відображення кількості обраних категорій
  
- Функціонал:
  - Завантаження списку всіх активних категорій
  - Завантаження поточних доступів виконавця
  - Додавання категорій до доступів
  - Видалення категорій з доступів
  - Масове оновлення доступів (збереження всіх змін одночасно)
  - Відображення секції тільки для ролі EXECUTOR
  
- API інтеграція:
  - GET /categories?is_active=true - отримати всі активні категорії
  - GET /users/{user_id}/category-access - отримати поточні доступи
  - PUT /users/{user_id}/category-access - оновити доступи (масово)
  
- Валідація та UX:
  - Попередження при відсутності доступів (виконавець не зможе бачити звернення)
  - Підтвердження при масовому видаленні доступів
  - Індикатор завантаження при збереженні
  - Повідомлення про успішне збереження
  - Обробка помилок API

## Визначення готовності (DoD)
- Секція "Доступ до категорій" відображається тільки для виконавців
- ADMIN може переглядати поточні доступи виконавця
- ADMIN може додавати/видаляти категорії для виконавця
- Зміни зберігаються на сервері
- Після збереження список доступів оновлюється
- Валідації та попередження працюють
- Помилки API відображаються користувачу
- UI інтуїтивний та зручний

## Тести
- Відображення секції категорій для виконавця
- Відсутність секції категорій для ADMIN та OPERATOR
- Завантаження списку категорій
- Завантаження поточних доступів виконавця
- Додавання категорії до доступів
- Видалення категорії з доступів
- Масове додавання кількох категорій
- Збереження змін (PUT запит)
- Попередження при відсутності доступів
- Відображення помилки при невдалому збереженні
- Індикатор завантаження працює коректно
