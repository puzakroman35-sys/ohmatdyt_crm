# BE-015: Healthcheck та базове логування

Фаза: 1 (MVP)
Тип: Backend

## Мета
Надати healthcheck ендпоінт і налаштувати структуроване логування для API/воркерів.

## Скоуп
- GET /healthz для API: стан DB (ping)
- Перевірка з’єднання Redis у воркері (ok/log)
- Логування у stdout у форматі JSON (рівні info/warn/error)

## Визначення готовності (DoD)
- /healthz повертає OK з базовою інформацією
- Логи містять request-id/trace-id (опц.)

## Тести
- Тест /healthz (200 OK); імітація відсутності DB

## Залежності
- BE-001, BE-013

## Позначки
- Healthcheck, Logging
