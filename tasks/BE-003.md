# BE-003: Категорії та Канали (CRUD, (де)активація)

Фаза: 1 (MVP)
Тип: Backend

## Мета
Реалізувати довідники Category та Channel з CRUD-операціями й можливістю активувати/деактивувати записи.

## Скоуп
- Моделі: Category { id, name, is_active }, Channel { id, name, is_active }
- Ендпоінти (ADMIN): GET/POST/PUT, POST /{id}/deactivate, /{id}/activate
- Валідація унікальності name

## Контракти API (приклад)
- GET /categories?include_inactive=bool
- POST /categories { name }
- PUT /categories/{id} { name }
- POST /categories/{id}/deactivate
- POST /categories/{id}/activate
- Аналогічно для /channels

## Визначення готовності (DoD)
- CRUD працює; унікальність name перевіряється
- Деактивовані категорії/канали не доступні у виборі при створенні звернення

## Тести
- Створення/редагування/деактивація/активація, валідація унікальності

## Залежності
- BE-002 (для RBAC ADMIN)

## Позначки
- Directories, Admin, Validation
